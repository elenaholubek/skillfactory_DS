Итак, для проекта "Выбираем авто выгодно" лично понадобилось создать кучу дополнительных файлов, и, чтобы не забыть, для чего они были созданы, свои мысли я буду записывать сюда

0. Parsing_links.ipynb : наполняет папку LINKS ссылками на объявления Для одной марки должен быть один файл, например, "AUDI_links.csv" Использует файл "brands_soup.txt" из корневого каталога
	"brands_soup.txt": элемент "div class="ListingPopularMMM PageListing__popularMMM" 	id="popularMMM"" первый страницы сайта Для нового парсинга можно обновить
	LINKS: готовый контент, созданный "0. Parsing_links.ipynb" При желании можно обновить, не удаляя старую информацию

1. Parsing_content.ipynb : наполняет контентом папку CONTENT. Запись в файл происходит частями по 100 (настраиваемо) объявлений. Использует контент из LINKS Создает и использует ads_counts.json, startpages.json Вывод информации на экран дает возможность проверить корректность цены, у меня на устройстве, попавшем в черный список, парсились неправильные цены

2. Preprocessing.ipynb, 3. EDA.ipynb : здесь изменяются данные с использованием человеческой логики. Обработанные данные записываются в папку PROCESSED


4. Feature engineering.ipynb : подготовка данных к построению моделей Создает несколько датасетов, так как принять решение в по нескольким вопросам заранее было сложно Сохраняются данные в папку ML


5. CAR PRICE PREDICTION - ML.ipynb : построение моделей. Данные на вход и результаты в папке ML
